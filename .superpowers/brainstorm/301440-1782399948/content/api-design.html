API接口设计

RESTful API接口规划，通过GateServer统一转发

API接口列表

用户管理接口 (UMSServer)

接口 方法 说明
/user/register POST 用户注册
/user/login POST 用户登录
/user/profile GET/PUT 获取/更新个人信息
/user/list GET 用户列表（管理员）

车辆管理接口 (VehicleServer)

接口 方法 说明
/vehicle/list GET 车辆列表（支持筛选）
/vehicle/{id} GET 车辆详情
/vehicle POST 添加车辆
/vehicle/{id} PUT 更新车辆信息
/vehicle/{id} DELETE 删除车辆

租赁业务接口 (VehicleServer)

接口 方法 说明
/rental/create POST 创建租赁订单
/rental/{id}/pickup POST 取车登记
/rental/{id}/return POST 还车结算
/rental/{id}/renew POST 续租处理
/rental/list GET 租赁订单列表

财务接口 (FinanceServer)

接口 方法 说明
/payment/create POST 创建支付记录
/payment/{id} GET 支付详情
/invoice/generate POST 生成账单
/stats/overview GET 统计概览
/stats/revenue GET 收入统计

API设计原则

  • RESTful风格：使用标准HTTP方法（GET/POST/PUT/DELETE）
  • 统一响应格式：{ "code": 0, "message": "success", "data": {...} }
  • 认证机制：JWT Token，通过Authorization头传递
  • 错误处理：统一错误码和错误信息
  • 分页支持：列表接口支持page和pageSize参数
