数据库设计

车辆租赁管理系统核心数据模型

实体关系图 (ER图)

用户相关表

users (用户表)
  • id (PK)
  • username
  • password_hash
  • phone
  • email
  • id_card
  • driver_license
  • role (customer/staff/admin)
  • status (pending/active/banned)
  • created_at
user_profiles (用户详情)
  • user_id (FK)
  • real_name
  • gender
  • birth_date
  • address
  • avatar_url

车辆相关表

vehicles (车辆表)
  • id (PK)
  • plate_number
  • brand
  • model
  • color
  • year
  • mileage
  • status (available/rented/maintenance)
  • daily_rate
  • image_url
vehicle_documents (证件表)
  • vehicle_id (FK)
  • driving_license_url
  • insurance_url
  • inspection_url
  • purchase_date
  • insurance_expiry

租赁业务表

rental_orders (租赁订单)
  • id (PK)
  • user_id (FK)
  • vehicle_id (FK)
  • start_date
  • end_date
  • actual_return_date
  • status (pending/active/completed/cancelled)
  • deposit
  • total_cost
maintenance_records (维保记录)
  • id (PK)
  • vehicle_id (FK)
  • type (repair/maintenance)
  • description
  • cost
  • start_date
  • end_date
  • status (pending/in_progress/completed)

财务相关表

payments (支付记录)
  • id (PK)
  • order_id (FK)
  • amount
  • type (deposit/rental/penalty)
  • method (cash/card/wechat)
  • status (pending/paid/refunded)
  • paid_at
invoices (发票/账单)
  • id (PK)
  • order_id (FK)
  • invoice_no
  • amount
  • items (JSON)
  • generated_at

数据库设计原则

  • 规范化设计：遵循第三范式，减少数据冗余
  • 索引优化：为常用查询字段创建索引（如plate_number, user_id, status）
  • 外键约束：确保数据完整性，级联更新/删除
  • JSON字段：灵活存储扩展信息（如车辆配置、账单明细）
  • 时间戳：所有表包含created_at和updated_at字段
