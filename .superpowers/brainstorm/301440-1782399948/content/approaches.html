系统架构方案对比

基于你的技术栈选择（C++后端 + Node.js/Vue3前端 + MySQL/Redis + Docker），以下是三种架构方案

A

简化微服务架构 (推荐)

架构：3-4个C++微服务 + Vue3前端

服务划分：

  • GateServer - HTTP网关，请求路由
  • UMSServer - 用户管理服务
  • VehicleServer - 车辆档案+租赁业务+维保管理
  • FinanceServer - 收费结算+统计报表

优势：复用OxyTeamTasks架构，展示分布式系统能力，符合课程设计要求

劣势：开发复杂度较高，需要处理服务间通信

B

单体C++后端 + Vue3前端

架构：1个C++后端服务 + Vue3前端

服务划分：

  • MainServer - 统一处理所有业务逻辑
  • Vue3 Frontend - 前端界面

优势：开发调试简单，部署方便，减少基础设施依赖

劣势：无法展示微服务架构能力，与OxyTeamTasks架构差异大

C

Node.js后端 + Vue3前端

架构：Node.js/Express后端 + Vue3前端

服务划分：

  • Express Server - RESTful API服务
  • Vue3 Frontend - 前端界面

优势：全JavaScript技术栈，开发效率最高，与MailerServer技术一致

劣势：无法复用C++代码，与OxyTeamTasks架构完全分离

我的推荐：方案A - 简化微服务架构

理由：

  • ✅ 最大程度复用OxyTeamTasks的C++微服务架构和代码
  • ✅ 符合课程设计要求（展示软件工程全流程）
  • ✅ 通过Docker容器化展示现代部署能力
  • ✅ 3-4个服务的规模在4周内可完成
  • ✅ 前端使用Vue3 + Element Plus，与OxyTeamTasks前端技术一致
